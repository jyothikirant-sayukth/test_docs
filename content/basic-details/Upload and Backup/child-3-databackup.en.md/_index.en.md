# <u>Data Backup</u>
If you are unable to upload the records due to low internet connection or no internet, you can use this feature.<br>

{{< notice "note" >}}
  <b>Sender :</b> The person who wants to transfer the data to another mobile.<br>
  <b>Receiver :</b> The person who will receive and upload the received data.
{{< /notice >}}

{{< mediacms url="https://media.panchayatseva.in/embed?m=6CsmjsnIR" title="Panchayat Seva Video" >}}

## Navigation

1. Open the <b>Menu</b> on the top left corner of the Dashboard screen.<br>
{{< image src="images/House_images/dashboard_screen.jpg" alt="Alt text" width="250" height="580" class="img-center">}}<br>
1. From the <b>Side Menu</b>, select the <b>Survey Data Backup.</b> <br>
{{< image src="images/navigation_images/s_data_backup_mb.jpeg"alt="Alt text" width="250" height="500" class="img-center">}}
1. After clicking on the Survey Data Backup, you will be moved to the Survey Data Backup screen.


### Details Section
#### Hotspot Connection Setup
1. The Receiver should turn on the Mobile Hotspot.
1. The Sender should turn on the Wi-Fi and connect to the Receiver's Mobile Hotspot to start the Data Backup.<br>
#### Device Connection
1. Once both mobiles are connected, the Sender's and Receiver's screens will show a simple guide on how to back up survey data.<br>
   + <b> Sender :</b> Sender can see the <b>Send Data</b> button.<br>
   + <b>Receiver :</b> Receiver can see the <b>Receive Data</b> button.<br>
   {{< image src="images/common_images/send_receive_button.jpg"alt="Alt text" width="550" height="500" class="img-center">}}<br>
   {{< notice "note" >}} The <b>Send Data</b> button will be visible only when the Sender is connected to the Receiver's Hotspot. {{< /notice>}}
1. The Sender should click on the <b>Send Data</b> button to send the Data.<br>
1. The Receiver should click on the <b>Receive Data</b> button to receive the Data.<br>
#### IP Address Setup Process
1. Initially, the connection status on both the devices will displayed as <b>Not Connected</b>.<br>
{{< image src="images/common_images/qr_code_screens.jpg"alt="Alt text" width="550" height="500" class="img-center">}}
1. The Sender can receive the Receiver's IP Address and Port Number using one of the following methods:
    ##### QR Code Scan
      1. The Sender should scan the QR code from the Receiver's Screen then the IP Address, Port Number will displayed automatically.
      1. The connection status will be updated to <b>Connected</b> on both the devices.
    ##### Manual Entry
      1. The Sender manually enters the Receiver's IP Address and Port Number displayed on the Receiver's mobile.
      1. Click on the <b>Connect</b> button, then the connection status will be updated to <b>Connected</b> in both the devices.<br>
{{< image src="images/common_images/connected_status.jpg"alt="Alt text" width="550" height="500" class="img-center">}}<br>
   {{< notice "note">}} If you click on the <b>Connect</b> button without receiving the IP Address and Port Number then an message will be displayed at the bottom of the screen as <b>" Please Provide Valid Receiver IP & Port Number "</b>.{{< /notice >}}<br>
#### Initiating Data Transfer
1. Once Sender and Receiver's Mobiles are connected, both the screens will displays as follows :
   + <b>Receiver's Screen : </b>The Receiver's Screen will display the Receiver Status as <b>Ready to Receive</b> and remains on the same screen, waiting to receive the data from the Sender.
   + <b>Sender's Screen : </b> The Sender's Screen will display the <b>Send Data</b> button and click on it to send the Data.
1. After clicking on the <b>Send Data</b> button by the Sender, the Sender Screen will move to the Next Screen in which it displays the Survey Data Backup Tutorial, Connection Status and Data Backup List.<br>
{{< image src="images/advertisement_images/databackup_images/ad_backup_list.jpeg"alt="Alt text" width="250" height="900" class="img-center">}}<br>
#### Data Transfer1. The Sender clicks on the Property from the Data Backup List, which moves to the that Property Data Backup Screen and displays the Send Backup Data section.
   1. <b>Backup Data :</b> To Backup the Available Records.
      1. <b>Available Records :</b> The Number of Records which are available in Sender's Mobile. 
      2. <b>Backup Records :</b> The Number of Records which are available to 
      transfer to another mobile.
      3. <b>Last Data Backup : </b> It will displays the exact time and date of the Sender's recent backup.
      4. <b>Send Data :</b> It is a button to transfer the Back up Records to the Receiver's Mobile.<br>
  {{< image src="images/advertisement_images/databackup_images/ad_send_button.jpg"alt="Alt text" width="300" height="250" class="img-center">}}<br>
1. By clicking on the <b>Send Data</b> button, the Data Backup records will be back up and updates the Backup Records count, and displays a message as <b>" Successfully Backup the Records "</b>.
      1. If you click on the <b>Send Data</b> button again with no new records available, the following message displays a message as <b>" No latest Records available to Send "</b>.<br>
2. Upon clicking the Send button, the following messages display as :
      + Sender's Mobile : <b>" File Send Successfully "</b>
      + Receiver's Mobile : <b>" Received Successfully "</b>
3. Once the Receiver receives all the Data Backup Files, the Sender and Receiver can disconnect the connection between the two devices.
   1. If the Sender and Receiver clicks on the <b>Disconnect</b> button, a warning dialogue box will display.
      + Click <b>" Yes "</b> button, to disconnect the connection.
      + Click <b>" No "</b> button,  to remain the connection active.<br>
   {{< image src="images/common_images/disconnect_alert.jpg"alt="Alt text" width="550" height="500" class="img-center">}}<br>
{{< notice "note" >}}
Editing and Deleting cannot be done for Backup Advertisement Records.{{< /notice >}}
